#+title: Parse bless logs with cf-ci
#+author: Rohit Goswami

* Overview

=bless= logs can be parsed by =cf-ci= to extract build errors. This is useful
for automated error analysis of local builds.

* Clean output for piping

Use =--no-timestamp= to produce output that =cf-ci= can parse cleanly:

#+begin_src bash
bless --no-timestamp -- make 2>&1 | cf-ci parse
#+end_src

* Parse from gzip files

=cf-ci= can also parse gzip log files directly:

#+begin_src bash
bless --label build -- make -j8
cf-ci parse build_*.log.gz
#+end_src

* JSON lines mode

For structured processing:

#+begin_src bash
bless --format=jsonl -- make 2>&1 | jq 'select(.level=="WARN" or .level=="ERROR")'
#+end_src

* Separate stdout and stderr

Use =--split= to get separate log files, then parse only stderr:

#+begin_src bash
bless --split --label build -- make -j8
cf-ci parse build_*_stderr.log.gz
#+end_src
